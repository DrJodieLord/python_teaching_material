(2) Jupyter Basics¶

Opening Jupyter Notebooks:

  • Notebooks are opened from your browser

  • The Jupyter browser directory space mirrors your local folder space - navigate to where your file is located from the browser space to open

Running and saving code:

  • Press "Run" button to run the code

  • Press "+" button to add more cells

  • Press the button to save your notebook (but Jupyter will save as you go also!)

Clearing content:

  • To clear current output (cell within): Cell --> Current Outputs --> Clear
  • To clear all output (entire notebook): Cell --> All Output --> Clear

Stopping code:

  • If you notice your notebook has frozen or code is taking unusually long - press the button

Edit vs command mode

  • When working within a cell, it will turn green - here you are in edit mode

  • You can exit edit mode by either:

    • clicking somewhere outside your cell
    • pressing escape
    • running your code

  • When you are on a cell but not in edit mode (known as "command mode") - the cell will turn blue

Keyboard shortcuts

  • a, b = insert cell above, below

  • x, c, v, z = cut, copy, paste, undo

  • m, y = change cell to markdown, code

  • s = save

  • f = find/replace

  • o = toggle output

  • h = help

  • p = command palette

  • Up and down arrows on keyboard = navigate from one cell to the next

**NOTE** : shortcuts only works in Command mode, so hit Esc first if you're in edit mode!

(2) Python Basics¶

Variables

  • A variable stores some sort of data

  • The value of a variable is assigned using a single = E.g. my_variable = hello world

  • If you then print that variable, you will be shown its value

Try below

Variable Names

  • Variables can be reassigned as much as you like.

  • You can name your variables whatever you like,

    • BUT: Avoid names that have a meaning in python: (e.g. the word 'list', or words like 'for' 'in' 'type' - these words go green when you type them)

  • Make names informative (so you know what the variable refers to).

  • NoSpacesAllowed - variable names must either be in CamelCase or use_underscores

Comments

The # statements within the code chunk above refer to "comments". These are bits of code which aren't evaluated by python when you run that section of code. They are EXTEMELY useful for:

  • Allowing you to remember what your code does when you come back to it at a later date.
  • Other people understanding the code you have produced.

Get into the habit of commenting your code!

You can never have too many comments!

Within your final assessment, you will be awarded marks for comments as well as the code produced.

Built in functions

  • Python functions let you do something.
  • Python has a whole host of built in functions - ready to go.
  • print() is an example of a built in python function - it does what it says on the tin: allows you to print your code output into the notebook.
  • Built in functions can be identified by a name followed by parentheses (e.g. print(), mean(), len() etc).
  • Arguments of a function go within the parentheses, and multiple arguments can be seperated by a comma (e.g. print(x, y, z).

  • Functions can be nested within each other - e.g. if you wanted to print the length of x, you can: print(len(x)).

  • At the more advanced stages of your python journey, you may dabble with creating your own functions, but for now, let's keep it simple.

Data types

  • String = text --> Indicated by wrapping the text in quotes (, or ) - E.g. hello world

  • integer = whole number (1, 2, 100, 183467218346234)

  • float = decimals (0.1, 1.4234, 2.54)

  • boolean = True/False

Slicing¶

Any lists of sequences can be "sliced" - that is, we can cut out specific sections of the sequence that we are interested in. This is possible for any variable which contains a sequence, including:

  • Lists
  • Tuples
  • Strings

Slicing can be achieved using the subscript notation: []

Discussion

  • Run the below code and take a look at sliced sequence outputs. What do you notice?
    • You may expect slicing the list example at index 3 would return the third item in the list, and that the 1st index of "hello" would be "h". However, as we see below, index 3 returns item 4 in the list example, and index 1 returns item 2 in the string example. This is because indexing starts at 0, so item 1 will be assigned to index 0, item 2 to index 1 and so on. Bare this in mind when indexing your code!
      </br>

Discussion

  • Can you think of when a tuple might have advantage / be of use over lists and vice versa?

Possible answers include:

  • Tuples are faster than lists because they contain contant values. If you're working with large arrays of data and performing computationally heavy tasks, this can be an advantage if you know the values aren't likely to change.

  • Tuples can be used as dictionary keys (see below). This is not possible for lists, as lists are mutable.

  • Lists allow you to dynamically update information on the fly. So if you know information is likely to change over time or that new information is going to be coming in in later waves that you want to add to your set of values, then lists are the way to go.

Dictionaries ("Dicts")¶

  • Python allows you to create "data dictionaries", whereby you can assign keys to values.
  • Dicts are in part mutable and in part immutable: keys are immutable, and they are assigned to values which are mutable (i.e. they can be changed).

  • Each dict key must be unqique and is associated with a corresponding value pair.

  • There are a number of ways you can create a dict in python, some of examples are below:

In [56]:
### USING CURLY BRACKETS & x:y KEY VALUE PAIRS ###

patient_results_curly = {'Case1':674, 'Case2':442, 'Case3':445, 'Control1':992, 'Control2':995}


### FROM A LIST OF TUPLES ### 
patient_results_tuples = dict([('Case1', 674), ('Case2', 442), ('Case3',445),('Contro11', 992), ('Control2', 995)])


### FROM KEYWORDS AND VALUES ###

# Construct a dictionary from keywords and values
patient_results_keys = dict(Case1=674, Case2=442, Case3=445, Control1=992, Control2=995)


#These are all different ways of achieving the same result:
print(patient_results_curly)
print(patient_results_tuples)
print(patient_results_keys)

An alternative way to create dicts is also to "zip together" 2 seperate lists. The below is example code of how to do this. Don't worry too much about the knitty gritty of the code for now (it contains looping which we have not covered here), but know it is possible:

In [57]:
patients = ['Case1', 'Case2', 'Case3', 'Control1', 'Control2'] #Patient ID list
results = [674, 442, 445, 992, 995]  #Seperate list of results

zipped_example = {x for x in zip(patients, results)} #Zipping the two lists together - patient as key, results as values

print(zipped_example) #printing results into notebook

{('Case2', 442), ('Case3', 445), ('Control1', 992), ('Control2', 995), ('Case1', 674)}

    • In research, it is common to recieve through information to do with a sample ID or patient ID in different streams / different lists/documents etc. The ability to zip these all together in a single dict so that you can refer to all information that corresponds to a specific subject ID can come in very handy

Retrieving information in dicts

  • Because each value is assigned to a key - we can call on the key to return its corresponding value - much like we do with an index value in lists.
  • Keys can also be used to modify its paired value.
  • New values can also be added to the dict by assigning them to new keys.

See examples below
